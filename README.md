## 🛠 Stack technique & choix
Le projet est construit sur une architecture **MERN** (MongoDB, Express, React, Node.js).

* **Framework** : **React (Vite)**. Choisi pour sa rapidité au développement et sa gestion efficace de l'état du formulaire via les hooks.
* **Styling** : **Tailwind CSS**. Utilisé pour reproduire la maquette avec minutie grâce à ses classes utilitaires, permettant de gérer facilement le glassmorphism (transparence/flou) et le responsive.
* **Base de données** : **MongoDB Atlas**. Un choix orienté flexibilité pour stocker les documents de contact sans schéma rigide complexe.
* **Gestion des données** : **Mongoose**. Utilisé côté backend pour structurer les données envoyées et assurer la validation avant l'enregistrement en base.
* **Variables d'environnement** : **Dotenv**. Utilisé pour sécuriser les identifiants de connexion à la base de données.
